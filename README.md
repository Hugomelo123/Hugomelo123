🗂️ Operational Demos

These are not products. They are structured demonstrations of operational thinking — built to show how I diagnose a problem, map the information flow, and propose a concrete structure.
Each one was built around a real problem I observed directly.

<br>
<details>
<summary><b>🏗️ ChantierOps — Construction Site Reporting</b></summary>
<br>
The problem
End of day: site manager calls 8 team leaders. Gets vague answers. Hangs up with a rough idea of progress — nothing written, nothing precise. If something went wrong, he finds out the next morning.
The pattern
8 sites → 8 calls → 0 written records → 0 real visibility
The demo
Team leaders send a WhatsApp message — same behaviour they already have.
Site manager gets a real-time dashboard instead of 8 phone calls.
Stack
React 19 TypeScript Node.js Express PostgreSQL Tailwind Twilio API
🔗 Live demo — chantierops-production.up.railway.app
</details>
<br>
<details>
<summary><b>⚡ SolarOps — Installation Pipeline (Luxembourg)</b></summary>
<br>
The problem
Leads in Excel. CREOS files forgotten for weeks. Quotes sent, never followed up. Technicians dispatched by phone every morning.
Not a technology problem. A visibility and coordination problem.
The pattern
Information exists → but disconnected → decisions made blind
The demo
Full pipeline from lead to installation. Every stage visible. Blockers surfaced before they become delays.
Lead → Visite → Devis → CREOS → Installation → Raccordement → Terminé
Stack
React 19 TypeScript Express Tailwind pdf-lib
</details>

<details>
<summary><b>💶 LuxPayroll — Payslip Verification</b></summary>
<br>
The problem
Luxembourg payslips are complex. Most employees have no way to verify them independently.
The result
This demo was used on a real payslip and identified confirmed discrepancies.
Full credibility cycle:
Problem observed → Demo built → Real error detected
The only demo with documented real-world use.
Stack
NestJS Next.js PostgreSQL TypeScript
</details>
